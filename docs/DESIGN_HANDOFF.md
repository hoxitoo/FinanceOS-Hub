# FinanceOS-Hub — Design Handoff Brief

> Пакет для Claude Design. Цель: оценить и развить концепцию UI-редизайна
> «Мерцание» (Shimmer / биолюминесценция) — анимации, новое отображение
> карточек, «светлячки и загадочность», переключаемая тема в настройках.
> **Сейчас интересует только дизайн-ревью и развитие идеи, не реализация.**

---

## 1. Что это за приложение

**FinanceOS-Hub** — Android-приложение для личных финансов, offline-first.
Читает SMS/пуши от банков → автоматически парсит и категоризирует операции →
показывает аналитику и поведенческие инсайты. Полностью без сервера, все данные
локально (Room/SQLite).

- **Платформа:** Android (Kotlin + Jetpack Compose, Material3)
- **Min SDK 26, Target 34**
- **Архитектура:** Clean Architecture + MVVM + Hilt + Room + Coroutines/Flow
- **Аудитория:** русскоязычная (вся локализация — RU); поддержка мульти-валюты
  (RUB, USD, EUR, KGS) — есть пользователи из Кыргызстана (МБанк)
- **Тон продукта:** строгий, «финтех-премиум», тёмная тема, спокойствие и доверие,
  тактильные карты банков, тщательная типографика с tabular-nums.

### Поддерживаемые банки (11 парсеров)
Сбербанк, Т-Банк, ВТБ, Альфа-Банк, Газпромбанк, Райффайзен, Росбанк, Открытие,
МТС Банк, Почта Банк, Россельхозбанк, + МБанк (KG, мульти-валюта).

## 3. Дизайн-система (ТЕКУЩАЯ — что нельзя ломать по смыслу)

### Цвета (`FosColors.kt`) — тёмная тема, единственная
```
Фоны
  Background    #0A0D12   (почти чёрный, холодный)
  Surface       #111620
  Surface2      #181E2A
  Border        #1E2738
  BorderStrong  #3D4860   (поля ввода)

Акценты — СТРОГО СЕМАНТИЧЕСКИЕ
  Positive  #4DFFA0   доход, рост, успех, savings — ТОЛЬКО хорошее
  Negative  #FF6B6B   расход, тревога, превышение — ТОЛЬКО плохое
  Warning   #FFB84D   предупреждение, прогресс 70–90%
  Info      #4D9FFF   информация, ссылки

Текст
  TextPrimary    #E8ECF4
  TextSecondary  #7A8499
  TextMuted      #5A6478
  TextDark       #3A4358
```

### Типографика (`FosType.kt`)
- `HeroAmount` 34sp ExtraBold, letterSpacing −1.4, **tnum**
- `HeroLarge` 40sp Black / `HeroMinimal` 42sp ExtraBold / `HeroAmountMulti` 26sp (3+ валют)
- `ScreenTitle` 22sp ExtraBold
- `CardAmount` 16sp ExtraBold **tnum**
- `TxMerchant` 13.5sp SemiBold / `TxAmount` 13.5sp Bold **tnum**
- `Label` 11sp / `Micro` 10.5sp / `SectionCap` 11sp Bold letterSpacing 1.2
- **Правило: любой числовой/денежный Text → `fontFeatureSettings = "tnum"` (tabular-nums)**

### Размеры (`FosDimens.kt`)
- ScreenPadding/CardPadding 16dp, CardGap 10dp, SectionGap 20dp
- RadiusCard 18dp, RadiusChip 20dp, RadiusButton 12dp, RadiusBar 5dp
- IconSize 32/36/40dp, NavBar pill 54×28 r14

### Бренд-цвета банков (`BankColors.kt`)
Каждый банк → своя `BankBrand(bg, onBg)` (Сбер #1A9F29, Т-Банк #FFDD2D, ВТБ
#009FDF, Альфа #EF3124, и т.д.). Используются как фон объёмной карты.

### 5 НЕРУШИМЫХ правил дизайна
1. `Positive #4DFFA0` = доход/успех/savings ТОЛЬКО
2. `Negative #FF6B6B` = расход/ошибка/превышение ТОЛЬКО (суммы расходов в TransactionRow — обязательно Negative)
3. Все денежные/числовые `Text` → tabular-nums
4. `InsightCard` — только цветная левая граница, БЕЗ иконки внутри; цвет = severity
5. Net Worth отрицательный → Negative

### Семантика перевода (важно для цвета)
Операция-перевод (`TRANSFER`) рендерится **нейтрально** (`TextPrimary`, «↔ сумма»,
никогда не красный/зелёный). Расход — красный, доход — зелёный.

## 4. Хранение и доменные нюансы (контекст для дизайна цифр)
- Суммы хранятся как `Long` копейки (×100), в Double — только в `FosFormatter`
- Отрицательные копейки = расход, положительные = доход
- `FosFormatter` — RU-локаль, неразрывный пробел как разделитель тысяч,
  `currencySymbol()` для RUB/USD/EUR/KGS, `dayLabelYear()` («19 декабря 2025»)
- Финансовая оценка здоровья 0–100 (4 столпа: savings/stability/mandatory/cushion)
- Поведенческие архетипы: Плановик / Импульсивный / Гурман / Экономный / Путешественник

---

## 5. Что сейчас НЕ хватает с точки зрения визуала
- Только тёмная тема, нет светлой и нет вариативности
- Карты банков статичны (градиент + бейдж), без движения/глубины
- Числа меняются мгновенно, без анимации перехода
- Переходы между экранами — дефолтные
- Нет «настроения» / атмосферы, продукт строгий, но эмоционально нейтральный

## 6. ИДЕЯ ДЛЯ РЕВЬЮ: тема «Мерцание» (Shimmer / биолюминесценция)

> Хочется «масштабное дополнение UI»: новые анимации, новое отображение карточек,
> **«светлячки и загадочность»**, и чтобы это включалось отдельной темой в
> настройках (Стандарт ↔ Мерцание). Атмосфера: глубокий океан / биолюминесценция /
> тайна, при этом без потери «премиум-финтех» строгости и читаемости цифр.

### Палитра-сдвиг (черновик)
```
Background  #04040F   (почти чёрный с холодным синим)
Surface     #0A0A20 / #0D0D25
Glow 1      #00FFD1   (циан, биолюминесцентный)
Glow 2      #7B2FFF   (глубокий фиолет)
Glow 3      #FF6B9D   (глубоководный розовый, «медуза»)
Positive / Negative — те же #4DFFA0 / #FF6B6B, но «светятся» сильнее
```

### Концепции (черновые направления — нужен взгляд дизайнера)
1. **Светлячки (firefly particle system)** — 15–25 крошечных точек (2–4dp) на
   hero-блоке Dashboard и на Analytics. Пульсация прозрачности 0.3→1.0→0.3,
   медленный дрейф по Безье (20–40с цикл). Pure Canvas/DrawScope, без зависимостей.
2. **Glass morphism карточки** — фон `linearGradient(#FFFFFF0A → #FFFFFF03)`,
   1dp граница #FFFFFF15, radius 24dp, мягкое свечение (shadow ambient #00FFD180).
3. **Голографические карты банков** — sweep-градиент 0°→360° на 15% прозрачности +
   tilt-параллакс на касании (±8°, spring обратно).
4. **Анимированные счётчики чисел** — баланс «прокручивается» (AnimatedContent
   slide+fade), ScoreRing считает от 0 при появлении.
5. **«Дыхание» hero-суммы** — едва заметный масштаб 0.998→1.000, 3с цикл, мягкое свечение текста.
6. **Переходы экранов** — fadeIn + scaleIn(0.97) «проявление из темноты».
7. **«Созвездие» категорий** — альт. визуализация: категории-узлы, связи-линии,
   медленная орбита (60с).

### Технический план включения
- `ThemeVariant` (sealed): `Standard` (текущая) / `Shimmer`
- `UserPreferences.THEME_VARIANT` (string-ключ DataStore)
- Settings → секция «ГЛАВНЫЙ ЭКРАН» → ряд «Тема» с чипами Стандарт/Мерцание
- `FosTheme` читает префу и подменяет colorScheme
- Новые composable: `ParticleSystem`, `GlassCard`, `ShimmeringBankCard`

## 7. Вопросы к Claude Design (что хочется получить)
1. Насколько концепция «биолюминесценция» совместима с доверием к финтеху? Где грань
   между «загадочно/премиум» и «несерьёзно/игрушка»?
2. Как сохранить читаемость денежных сумм (tnum, контраст) поверх свечения/частиц?
3. Стоит ли «Мерцание» быть полной альтернативной темой или слоем-надстройкой
   (частицы/свечение) над текущей тёмной базой?
4. Какие из 7 концепций дают максимум «вау» при минимуме риска для usability/перформанса?
5. Палитра: предложить выверенную shimmer-палитру (фоны, surface, 2–3 glow-акцента),
   совместимую с семантикой Positive/Negative.
6. Микро-анимации: что уместно по умолчанию, а что прятать за «reduce motion».
7. Новое отображение карт банков — конкретные направления (глубина, материал, блик, движение).

## 8. Технические ограничения (для реалистичности предложений)
- Jetpack Compose, Material3, без тяжёлых граф-движков; анимации — `Canvas`,
  `infiniteTransition`, `animateFloat`, `AnimatedContent`, `graphicsLayer`.
- Compose пока без нативного blur на min SDK 26 → стекло эмулируется слоями
  полупрозрачных surface, не реальным размытием.
- Offline-first, без сети; всё рисуется на устройстве — следить за перформансом
  (частицы/бесконечные анимации не должны жечь батарею).
- Min SDK 26 — некоторые эффекты (RenderEffect.blur) доступны только с API 31+,
  нужен graceful fallback.
